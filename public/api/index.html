Parameters

All endpoints use the same parameter structure

Required

user_email
string Required
Email address - your Aquamark account email (retail) or end user's email (wholesale)
file
file Required
One PDF file (max 25MB)

Recommended

state
string Recommended
For state compliance disclaimers – accepts state names or abbreviations (e.g. "CA", "California"). Returned in response header. To store or display this value, create a custom field in your CRM called Compliance and push the value there automatically.

Advanced Tracking

Tamper-resistant QR code on every page with embedded access information
Power up your QR codes

These fields act as digital breadcrumbs that show exactly who had access to each file — lender, salesperson, processor.

They only apply if your CRM enforces curtaining, where users can only access records assigned to them. Curtaining is strongly recommended as a best practice.

salesperson
string Optional
Internal salesperson associated with the deal. Typically your CRM account owner or assigned rep.
processor
string Optional
Team member responsible for preparing and submitting files. Only useful if you assign dedicated processors.
lender
string Optional
Users submit a separate version of the file for each lender, then delete them after submission to reduce storage costs. Most set up a simple automated deletion—using tools like timed scripts, CRM schedulers (e.g. Apex Scheduler in Salesforce), or external automation platforms like Zapier or Make.

Postman Example

Postman Setup

POST {{endpoint}} Headers: Authorization: Bearer YOUR_API_KEY Content-Type: multipart/form-data Body (form-data): user_email: john@example.com file: [Select File] statement1.pdf state: CA salesperson: Jack Dawson processor: Rose DeWitt lender: OnDeck Endpoints: Retail: https://aquamark-decrypt.onrender.com/watermark CRM: https://aquamark-cloudsquare.onrender.com/watermark OCR: https://aquamark-ocr.onrender.com/watermark

Response

All endpoints return a protected PDF file with optional compliance header:

Content-Type: application/pdf X-State-Disclaimer: CFL license and Disclosure required.

Workflow

  1. Choose the appropriate endpoint for your use case
  2. Send a POST request with your PDF and metadata
  3. Receive a protected PDF + optional state disclaimer header
  4. Save the PDF and push state disclaimers to your system

Important Notes

  • PDF only. One file per request across all endpoints
  • Retail users must upload logo in portal before using the API
  • Wholesale partners receive dedicated onboarding and integration support
  • Questions? Email hey@aquamark.io (retail) or partners@aquamark.io (wholesale)
