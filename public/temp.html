Simple API Integration

Every CRM is different. Your only job is to intercept the PDF download, send it to the Aquamark API, and return the processed file to the user.

API Endpoint

POST https://aquamark-lenders.onrender.com/watermark

Headers

Authorization: Bearer YOUR_API_KEY

Request Format

Use multipart/form-data with the following fields:

Field Type Required Description
file File âœ“ The original PDF to protect
name String âœ“ CRM user's full name (person downloading)
company String âœ“ Company name (e.g., "Acme Lending")

Response

Returns a 200 OK with the protected PDF as binary data. Client should treat it as a file download.
