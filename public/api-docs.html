cURL

curl -X POST "https://aquamark-decrypt.onrender.com/watermark" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F "user_email=john@example.com" \
  -F "file=@statement1.pdf" \
  -F "lender=Fora Financial" \
  -F "salesperson=Jack Dawson" \
  -F "processor=Rose DeWitt" \
  -F "state=CA" \
  --output "statement1 - protected.pdf"

If a valid state is provided and matches regulation rules, the response will include a compliance disclaimer as an HTTP header:

This value will not appear in the PDF file. If you would like to store the disclaimer in your CRM, you must extract this header and push it into a custom field (e.g. compliance_disclaimer) using your API client or integration workflow.
